Vue.js 中的 mixin 是一种重用代码的模式，它允许你在多个组件之间共享 Vue.js 组件的可复用功能。一个 mixin 对象可以包含任何组件选项。当组件使用 mixin 对象时，所有 mixin 对象的选项将被“混入”该组件本身的选项。

使用 mixin 的主要好处是可以将一些常用的代码提取出来，避免在多个组件中重复编写相同的代码，提高代码的可维护性和可复用性。

下面是一个使用 mixin 的示例：

// 定义一个 mixin 对象
var myMixin = {
  created: function () {
    this.hello()
  },
  methods: {
    hello: function () {
      console.log('Hello from mixin!')
    }
  }
}

// 定义一个使用 mixin 的组件
var Component = Vue.extend({
  mixins: [myMixin],
  created: function () {
    this.hello() // => "Hello from mixin!"
  }
})

var component = new Component() // => "Hello from mixin!"

在上面的示例中，我们定义了一个名为 myMixin 的 mixin 对象，其中包含一个 created 生命周期钩子和一个 hello 方法。然后，我们定义了一个使用 myMixin 的组件 Component，通过在组件选项中设置 mixins 属性来应用 mixin。最后，我们创建了一个 Component 的实例，并调用了它的 hello 方法。由于组件使用了 mixin，所以它会输出 "Hello from mixin!"。

需要注意的是，当组件和混入对象含有同名选项时，这些选项将以恰当的方式进行“合并”。例如，data对象在内部会进行递归合并，并在发生冲突时以组件的数据优先。对于大多数值为对象的选项，例如 methods、components 和 directives，将采用同样的合并策略。当组件和混入对象含有同名方法时，将采用组件的方法优先。
